Part 2 — CFE Module

What Is the Configurable Form Engine?

A single generic JavaScript web resource that drives model-driven app form behaviour from Dataverse configuration data — no per-table JavaScript ever again.

🚫

Problem it solves

Every table had its own custom JS for show/hide, required fields, defaults. Impossible to maintain at scale.

⚙️

How it works

Register one web resource on any form's OnLoad. All behaviour is driven by rows in 4 config tables — zero code per table.

♻️

Org-wide reuse

Onboard a new table in 15 minutes by adding config rows. No developer, no deployment, no solution export.

🔒

Safe by design

Guard rails: hidden fields never block saves, defaults never overwrite existing values, conditions re-evaluated on change.

Part 2 — CFE Module

8 Features — All Config-Driven

👁️

1. Visibility

Show or hide any field, tab, or section on Create / Update / Both. Hiding auto-clears required level.

⚠️

2. Requirement

Make fields Required or Optional — conditionally (e.g. only when Status = Active).

✏️

3. Editability

Set fields read-only or editable. E.g. Project Code locked after first save — no code needed.

📅

4. Default Values

Static text/number, Today, Current User, Lookup. Applied only when field is empty.

🔐

5. Value Locking

Lock When Set (can't change after entering) or Lock After Save (locked once record exists).

🎭

6. Role-Based Visibility NEW

Show/hide field, tab, or section by security role. ANY-match; backward-compatible with rules that have no roles.

✅

7. Field Validation NEW

7 built-in types (alpha, numeric, email, date range, max length, regex) with instant inline notifications.

🔒

8. Form-Level Lock NEW

Disable every control on the form — condition-gated. One rule row locks an entire form. Combine with roles to unlock for admins.

All conditions support 9 operators: Equals · Not Equals · Contains · Does Not Contain · Is Empty · Is Not Empty · In · Greater Than · Less Than — AND / OR grouping · Role gate on any rule

What Is a Token?

A token is the smallest unit of text the AI processes — roughly ¾ of a word or 4 characters. Every word you type and every word Claude generates is metered in tokens.

Text exampleTokensWhy
Hello1Short common word
Configurable3Longer = more tokens
mab_formrulecondition7Underscores split tokens
One line of JavaScript15–25Code is token-dense
CFE README (3,000 words)~8,000Loaded every session
Full 2-hour session history~400KContext grows with each turn
📥

Input tokens

Everything you send: your question + the full conversation history + any files/docs loaded. Priced lower — reading is cheaper than generating.

📤

Output tokens

Every word Claude writes back. Priced 5× higher than input — generation is compute-intensive (one token at a time, no parallelism).

⚡

Cached tokens

Repeated content (skill files, session history) stored in memory. Priced at 90% discount — the biggest efficiency lever in long sessions.

Part 5 — AI Cost Explained · Real-World Case Study

GitHub Copilot — Pricing Changed Because of Agentic AI

The world's largest AI coding tool abandoned flat pricing in 2025. The reason: agentic sessions consume 10–50× more compute than simple tab-completions. This is the proof point for everything on the previous slide.

❌ Old model (2022–2024)

  • Individual: $10 / month — unlimited
  • Business: $19 / seat / month — unlimited
  • One model (GPT-3.5 / GPT-4)
  • Completions + chat only
  • No agentic workflows

✅ New model (2025)

  • Free: 2,000 completions · 50 chats/mo
  • Pro: $10/mo → 300 premium requests
  • Pro+: $39/mo → 1,500 premium requests
  • Business: $19/mo → 300 premium req. included
  • Enterprise: $39/mo → 1,000 premium req.
  • Overage: $0.04 per additional request
WHAT IS A "PREMIUM REQUEST"?

A single interaction with a premium model (Claude Sonnet, GPT-4o, o1, Gemini Pro). A simple chat = 1 request. An agentic task (Copilot Workspace / Coding Agent) = 10–50 requests per task as it plans, edits files, runs tests, and iterates — all automatically.

⚡

Why flat pricing broke

A developer doing simple tab-completion = ~500 tokens/day. The same developer using Copilot Agent mode to refactor a module = 50,000–200,000 tokens per session. The cost ratio made unlimited pricing economically unsustainable.

🔮

Multiple models = multiple cost tiers

GPT-4o mini = cheap (base rate). GPT-4o / Claude Sonnet / o1 = premium. Each model has a different per-token cost. GitHub had to expose this to users rather than absorb it — exactly what our "3 pricing layers" slide describes.

🎯

The lesson for our CFE project

Claude Code's $20/month flat rate is today's equivalent of Copilot's old unlimited plan. Lock in reusable modules now. The move to consumption pricing is a question of when, not if.
